ANGIE: Agenda Inteligente

1.1 Propósito del Documento

Este manual técnico describe la arquitectura, componentes, tecnologías utilizadas y la lógica interna de la aplicación “ANGIE: AGENDA INTELIGENTE”. Está dirigido a desarrolladores, mantenedores y personal técnico que requiera comprender, modificar o extender la funcionalidad del sistema.

1.2 Alcance del sistema

ANGIE es una aplicación móvil desarrollada para Android que permite la gestión inteligente de actividades personales y académicas. Sus principales capacidades incluyen:

  • Autenticación de usuarios con validaciones robustas.
  • Gestión completa (CRUD) de eventos.
  • Sistema de notificaciones escalonadas y automatizadas.
  • Persistencia local mediante SQLite.
  • Interfaz moderna basada en Jetpack Compose.

2. ARQUITECTURA GENERAL

2.1 Patrón de diseño

La aplicación sigue el patrón MVVM (Model-View-ViewModel), lo que permite:

  • Separación clara de responsabilidades.
  • Mayor capacidad de prueba (testabilidad).
  • Ciclo de vida consciente de componentes Android.

ViewModelClases que exponen datos mediante StateFlow/LiveData y manejan la lógica de negocio

5.2 Recuperación de contraseña

El flujo técnico consiste en:

  1. Verificar que el usuario y correo correspondan al mismo registro en la tabla usuarios.
  2. Generar un código de verificación temporal (simulado o enviado).
  3. Validar el código ingresado por el usuario.
  4. Actualizar el campo password en la base de datos con la nueva contraseña (se recomienda hashing en futuras versiones).

7. SISTEMA DE NOTIFICACIONES INTELIGENTES

7.1 Algoritmo de recordatorios escalonados

ANGIE envía notificaciones en tres momentos clave antes de cada evento:

MomentoTiempo antes del evento
Primer recordatorio1 hora antes
Segundo recordatorio15 minutos antes
Tercer recordatorio5 minutos antes

7.2 Implementación técnica

Componentes involucrados:

  • AlarmManager: Programa las alarmas exactas usando setExact().
  • BroadcastReceiver: Recibe los intents de las alarmas.
  • NotificationManager: Construye y muestra la notificación.
  • SharedPreferences: Guarda un flag notificacion_enviada_{eventoId}_{momento} para evitar duplicados.

Flujo de programación:

  1. Al crear o modificar un evento, se calculan los tres tiempos de notificación.
  2. Para cada tiempo, se crea un PendingIntent único.
  3. Se registra en SharedPreferences que la notificación está pendiente.
  4. Al recibir la alarma, el BroadcastReceiver verifica que el evento siga vigente (fecha no pasada y no eliminado). Si corresponde, muestra la notificación y marca el flag correspondiente como true.

7.3 Persistencia tras reinicio del dispositivo

Se implementa un BroadcastReceiver que escucha el evento android.intent.action.BOOT_COMPLETED. Al reiniciar, consulta todos los eventos futuros del usuario, reprograma las alarmas necesarias (solo para eventos no pasados) y verifica los flags de SharedPreferences para reanudar correctamente el estado.

8. MANEJO DE FECHAS Y HORAS

8.1 Librería ThreeTenBP

Dado que las versiones antiguas de Android no soportan java.time, se utiliza el backport org.threeten.bp.*.

Clases principales: LocalDate, LocalTime, LocalDateTime.

8.2 Formatos soportados

  • Entrada del usuario: dd/MM/yyyy (convertido internamente a yyyy-MM-dd).
  • Almacenamiento en BD: yyyy-MM-dd (formato ISO).
  • Visualización: Se adapta según configuración regional.

10. FUNCIONALIDADES ADICIONALES

10.1 Obtención de eventos próximos (7 días)

Método en el repositorio:

SELECT * FROM eventos WHERE usuario_id = ? AND fecha BETWEEN ? AND ? ORDER BY fecha, hora
Los parámetros son: fecha actual y fecha actual + 7 días.

10.2 Eliminación automática de eventos pasados

Se ejecuta al iniciar la aplicación y cada vez que se consulta la lista de eventos. Borra eventos con fecha < LocalDate.now().

10.3 Prevención de nombres duplicados

Antes de insertar un nuevo evento, se consulta si ya existe un evento con el mismo nombre_evento y usuario_id.

12. CONCLUSIÓN TÉCNICA

ANGIE es una aplicación Android bien estructurada que combina tecnologías modernas (Jetpack Compose, MVVM) con soluciones robustas para persistencia (SQLite), manejo de fechas (ThreeTenBP) y notificaciones inteligentes (AlarmManager + BroadcastReceiver). Su arquitectura permite un mantenimiento ágil y una escalabilidad clara hacia nuevas funcionalidades. Las validaciones estrictas y el control de datos garantizan la integridad del sistema, mientras que el sistema de recordatorios escalonados ofrece una experiencia de usuario superior en comparación con agendas básicas.
